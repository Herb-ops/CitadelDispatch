# Source Map (Draft)

## Source tiers

### Tier 1 — Primary technical/governance sources

- Bitcoin Core and related GitHub repositories (PRs, issues, release notes)
- Bitcoin-Dev / Delving Bitcoin / relevant technical mailing lists
- BIPs repository and discussion threads
- Maintainer and contributor notes tied to direct artifacts
- Conference talks only when slides/code/references are available

### Tier 2 — Supporting infrastructure and measurements

- Reputable node/network telemetry dashboards with methodology
- Mempool and fee market data sources with transparent definitions
- Client release channels and changelogs

### Tier 3 — Commentary/context (lowest authority)

- Podcasts, long-form commentary, and social posts used only as directional leads
- Must never outrank Tier 1 evidence in claims hierarchy

## Citation rules inside each issue

- Every substantial claim should map to at least one Tier 1 or Tier 2 source.
- “Primary Sources” section must include direct links and one-line relevance notes.
- If evidence is ambiguous, label uncertainty explicitly.
- If camps disagree, cite strongest articulation from each serious side.

## Source freshness rules

- Revalidate unresolved controversies at least weekly.
- Do not carry forward old claims without checking for updates/merges/retractions.
